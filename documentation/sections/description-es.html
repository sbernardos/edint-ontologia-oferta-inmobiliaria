inmobiliaria: Descripción volver a índice

La ontología inmobiliaria define una única clase principal: Anuncio Inmobiliario (RealEstateListing), que representa una oferta de un inmueble en el mercado, ya sea para venta, alquiler o traspaso. Cada anuncio tiene un identificador único y describe el tipo de inmueble, el tipo de transacción inmobiliaria y su estado actual (activo o inactivo). Esta ontología reutiliza GeoSPARQL para el soporte geoespacial y se integra con la ontología del Catastro Español para vincular los anuncios con sus entidades catastales correspondientes. Los anuncios inmobiliarios se describen mediante tres propiedades de objeto: tipo de inmueble (propertyType), que indica la categoría del bien inmueble (vivienda, local comercial, oficina, nave industrial o solar) y tiene como rango el esquema de conceptos SKOS Usos de Inmuebles del Catastro Español; tipo de transacción (transactionType), que especifica si la oferta es una venta, alquiler o traspaso, y tiene como rango el esquema de conceptos SKOS Tipos de Transacción Inmobiliaria; y se refiere a entidad catastral (refersToCadastralEntity), que vincula el anuncio con su entidad catastral correspondiente, que puede ser una CadastralParcel (parcela catastral) para terreno sin edificar, o una CadastralProperty (inmueble catastral) para edificios o unidades inmobiliarias. Además, los anuncios se describen con dos propiedades de datos: identificador (identifier), que proporciona un identificador único para cada anuncio, y está activo (isActive), que indica si la oferta inmobiliaria se encuentra actualmente vigente (verdadero) o inactiva (falso).
