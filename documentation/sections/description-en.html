The real estate ontology defines a single main class: Real Estate Listing (RealEstateListing), which represents a property offer on the market, whether for sale, rent, or transfer. Each listing has a unique identifier and describes the property type, the type of real estate transaction, and its current status (active or inactive). This ontology reuses GeoSPARQL for geospatial support and integrates with the Spanish Cadastre ontology to link listings with their corresponding cadastral entities. Real estate listings are described through three object properties: property type (propertyType), which indicates the category of the property (dwelling, commercial local, office, industrial warehouse, or plot) and has as its range the SKOS concept scheme Spanish Cadastre Property Uses; transaction type (transactionType), which specifies whether the offer is a sale, rent, or transfer, and has as its range the SKOS concept scheme Real Estate Transaction Types; and refers to cadastral entity (refersToCadastralEntity), which links the listing to its corresponding cadastral entity, which can be a CadastralParcel (cadastral parcel) for unbuilt land, or a CadastralProperty (cadastral property) for buildings or real estate units. In addition, listings are described with two data properties: identifier (identifier), which provides a unique identifier for each listing, and is active (isActive), which indicates whether the real estate listing is currently active (true) or inactive (false).
